JavaSE基础 day08

JAVA
  • 学习

继承

抽象类

抽象：笼统，模糊，不具体。
功能细节无法具体的方法，可以定义成抽象方法。类同理。

抽象类的特点：

  1. 方法只要声明没有实现时，该方法就是抽象方法，必须被abstract修饰。抽象方法必须定义在抽象类中， 该类必须也被abstract修饰。
  2. 抽象类不可以被实例化。 因为调用抽象方法没意义。
  3. 抽象类的所有抽象方法必须由其子类全部覆盖后，该子类才可以实例化。否则，这个子类还是抽象类。

思考五个问题：

  • 抽象类中有构造函数么？
    • 有，用于给子类对象初始化。
  • 抽象类可以不定义抽象方法吗？
    • 可以的，但是很少见，目的就是不让该类创建对象。AWT的适配器对象就是这种类。通常这个类中的方法有方法体，但是却没有内容。void show(){};
  • 抽象关键字不可以和那些关键字共存？
    • private不行，原因是抽象方法必须被覆盖，私有之后子类不能访问该抽象方法。
    • static不行，原因是如果抽象方法是静态，这样就不需要创建对象了，直接类名调用，但是抽象方法没有内容所以调用无意义。
    • final不行，final的功能是修饰的类不能被继承，修饰的方法不能被覆盖。而抽象类中的抽象方法必须被覆盖。
  • 抽象类和一般类的区别。
    • 相同点：抽象类和一般类都是用来描述事物的，都在内部定义了成员。
    • 不同点：
      1. 一般类有足够的信息描述事物。
        抽象类描述事物的信息可能不足。
      2. 一般类中不能定义抽象方法，只能定义非抽象方法。
        抽象类中可以定义抽象方法，同时也可以定义非抽象方法。
      3. 一般类可以被实例化。
        抽象类不能实例化。
  • 抽象类一定是父类？
    • 是的。因为需要子类覆盖其方法后才可对子类实例化。

抽象类练习

雇员示例：

需求：
公司中程序员有姓名，工号，薪水，工作内容。
项目经理除了姓名，工号，薪水，还有奖金，工作内容。
对给出的需求进行数据建模。

分析：
在这个问题领域中，先找出涉及的对象。
通过名词提炼法。
程序员：

  • 属性：姓名，工号，薪水。
  • 行为：工作。

经理：

  • 属性：姓名，工号，薪水，奖金。
  • 行为：工作。

程序员和经理不存在继承关系，但是有共性内容。可以进行抽取，因为他们都是公司的雇员。将程序员和经理进行抽取，建立体系。

接口

当一个抽象类中的方法都是抽象的时候，这是可以将该抽象类用另一种形式定义和表示，就是接口 interface。也就是说接口中的方法都是抽象的。

对于接口中常见的成员：这些成员都有固定的修饰符。就算是不写public static final或者public abstract系统都会自动加上。但是为了阅读性强我们都写上。

  1. 全局常量：public static final
  2. 抽象方法：public abstract

由此得出结论，接口中的成员都是公共权限。

接口的实现

  • 类与类之间是继承关系，类与接口之间是实现关系。
  • 接口的方法必须全部实现，关键字：implements
  • 接口不可以实例化。只能由实现了接口的子类并覆盖类接口中所有的抽象方法后，该子类才可以实例化。否则这个子类就是一个抽象类。

接口-多实现

在java中不直接支持多继承，因为会出现调用的不确定性。所以java将多继承机制改良，在java中变成了多实现。
一个类可以实现多个接口。
继承中父类方法体都要实现，多继承会导致多个父类中重名方法不能实现。而多实现中没有方法体。

一个类在继承另一个类的同时，还可以实现多个接口。
接口的出现避免了单继承的局限性。

接口继承

java中接口支持多继承，因为接口中没有方法体，重名的方法均可覆盖。

接口特点

  • 接口是对外暴露的规则。
  • 接口是程序的功能扩展。
  • 接口的出现降低了耦合性。
  • 接口可以用来多实现。
  • 类与接口之间是实现的关系，而且类可以继承一个类中的同时实现多个接口。
  • 接口与接口之间可以有继承关系，可以多继承。

接口与抽象类的区别

相同点：都是不断向上抽取而来的。

不同点：

  1. 抽象类需要被继承，而且只能单继承。
    接口需要被实现，而且可以多实现。
  2. 抽象类中可以定义抽象方法和非抽象方法，子类继承后，可以直接使用非抽象方法。
    接口中只能定义抽象方法，必须由子类去实现。
  3. 抽象类的继承是 is a关系，在定义该体系的基本共性内容。
    接口的实现是 like a关系，在定义体系的额外功能。

举例：
犬按功能分类：导盲，搜爆，缉毒。

接口的应用

笔记本电脑使用，为了扩展笔记本的功能，但日后出现什么功能设备不知道。定义一个规则，日后出现的设备都符合这个规则就可以了。规则就是Java中的接口。
