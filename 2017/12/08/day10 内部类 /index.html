内部类

概述

  • 将一个类定义在另一个类里面，对里面那个类就称为内部类（内之类，嵌套类）。
  • 访问特点：
    • 内部类可以直接访问外部类中的成员，包括私有成员。
    • 而外部类要访问内部类中的成员必须要建立内部类的对象。

作用：
一般用于用于类的设计。
分析事物时，发现该事物A描述中还有事物B，而且这个事物B还在访问被描述事物A的内容，这时候就可以将事物B定义成内部类来描述。

内部类-修饰符

直接访问外部类中的内部类中的成员格式：

外部类名.内部类名 对象名 = new 外部类名().new 内部类名();

static 修饰内部类情况
如果内部类中有静态方法，那么该内部类必须定义成静态

内部类-细节

为什么内部类能直接访问外部类中的成员呢？
那是因为内部类持有了外部类的引用。外部类名.this。

匿名内部类

也就是内部类的简写格式。
前提：内部类必须继承或者实现一个外部类或者接口。
匿名内部类其实就是一个匿名子类对象。
格式：new 父类or接口(){子类内容}

应用

使用场景之一：当函数参数是接口类型时，而且接口中的方法不超过三个。可以用匿名内部类作为实际参数进行传递。
